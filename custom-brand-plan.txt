# Plan: Custom User-Supplied Brand for NDS

## Context

NDS today ships three pre-baked token brands (Crimson, Corporate, Sunset) plus a stylesheet brand (Foundation Day). Commit `47ec54f` moved shipped brands off runtime OKLCH to build-time hex — correct for predictability and zero-flash, but it leaves no path for an end user to plug in their own seed. Result: `NDS.Brand` is mostly useful for the docs site's internal demo; everyone else just adds a CSS override file.

This plan adds a fourth path: a **Custom** entry in the topbar switcher with a native color picker. The visitor picks a primary hex (secondary + tertiary optional); the entire system re-skins live via CSS relative-color `oklch(from var(--brand-primary) …)`. Seeds persist to `localStorage` and re-stamp pre-paint so saved-return is flash-free. Shipped brands stay untouched — this is purely additive.

The killer demo: a docs site that proves the token architecture from a single swatch. The killer feature for consumers: bring your own brand without forking SCSS.

## Approach

- **Reuse the runtime mixins already in `_oklch-template.scss`** (`brand-ramp`, `brand-alpha`) — they're the *runtime* counterparts to `brand-ramp-static` that shipped brands use. Same `$_ramp` curve, so visual ramp shape matches shipped brands tile-for-tile.
- **Custom is just another `data-brand` value** (`data-brand="custom"`). No fork of `NDS.Brand` machinery.
- **User-supplied seeds win via inline `style` on `<html>`**. CSS-derived defaults (hue-rotated from primary) handle the unset case.
- **`@supports` gates the entire custom block** — graceful degradation to DGA on engines without relative-color OKLCH.
- **Skip neutral-tint and dark-mode overrides for v1** — keeps scope tight; both are easy to add later.

### `_sass/brands/_brand-custom.scss` — full file

```scss
@use 'oklch-template' as oklch;

// ════════════════════════════════════════════════════════════════════════════
// CUSTOM BRAND — runtime OKLCH ramp from user-supplied seed(s).
//
// Unlike Crimson/Corporate/Sunset (build-time static hex), Custom derives its
// palette at runtime via `oklch(from var(--brand-primary) …)`. The picker writes
// inline style on <html> + localStorage; secondary/tertiary derive from primary
// via hue rotation if the user does not supply them.
//
// Gated by @supports — on engines without relative-colour OKLCH this block is
// inert and the user sees the DGA baseline.
// ════════════════════════════════════════════════════════════════════════════

@supports (color: oklch(from white l c h)) {
  :root[data-brand="custom"] {
    // Default derivations — overridden by inline style="--brand-secondary: …"
    // from the picker JS when the user supplies their own.
    --brand-secondary: oklch(from var(--brand-primary) l c calc(h + 150));
    --brand-tertiary:  oklch(from var(--brand-primary) l c calc(h - 30));

    @include oklch.brand-ramp('primary',   '--brand-primary');
    @include oklch.brand-alpha('primary',  '--brand-primary');
    @include oklch.brand-ramp('secondary', '--brand-secondary');
    @include oklch.brand-ramp('tertiary',  '--brand-tertiary', 500);
    @include oklch.brand-alpha('tertiary', '--brand-tertiary');

    // Patch: runtime mixin emits alphas of the *seed* (600 anchor), not of step 500.
    --colors-primary-500-alpha-10: color-mix(in srgb, var(--colors-primary-500) 10%, transparent);

    --background-brand-strong: var(--colors-primary-900);
    --background-brand-light:  var(--colors-primary-25);
    --img-overlay-color:       var(--colors-neutral-950);
    --nds-font-brand:          'Tajawal', sans-serif;
  }
}
```

### `_includes/head-inline-scripts.html` — extended stamp

```html
<script>
  (function() {
    var r = document.documentElement;
    var t = localStorage.getItem('nds-theme');
    if (t === 'dark' || t === 'light') r.setAttribute('data-theme', t);
    {%- if site.brand_switcher %}
    var b = localStorage.getItem('nds-brand');
    if (b) r.setAttribute('data-brand', b);
    if (b === 'custom') {
      var seeds = ['primary','secondary','tertiary'], s, k;
      for (var i = 0; i < 3; i++) {
        k = seeds[i];
        s = localStorage.getItem('nds-brand-' + k);
        if (s) r.style.setProperty('--brand-' + k, s);
      }
    }
    {%- endif %}
  })();
</script>
```

### `_js/nds-brand.js` — additions inside the IIFE

```js
// New constants near the top, alongside SWITCH_SEL / LINK_ID:
const SEED_PREFIX = 'nds-brand-';
const PICKER_SEL  = '[data-brand-seed]';
const RESET_SEL   = '[data-brand-seed-reset]';

function setSeed(name, value) {
    if (!value) return clearSeed(name);
    root.style.setProperty('--brand-' + name, value);
    try { localStorage.setItem(SEED_PREFIX + name, value); } catch (e) {}
}
function clearSeed(name) {
    root.style.removeProperty('--brand-' + name);
    try { localStorage.removeItem(SEED_PREFIX + name); } catch (e) {}
}

// Inside init(), after the existing click listener:
document.addEventListener('input', (e) => {
    const inp = e.target.closest(PICKER_SEL);
    if (!inp) return;
    const seed = inp.getAttribute('data-brand-seed');
    if (!seed) return;
    setSeed(seed, inp.value);
    if (_current !== 'custom') set('custom');
});
document.addEventListener('click', (e) => {
    const r = e.target.closest(RESET_SEL);
    if (!r) return;
    const seed = r.getAttribute('data-brand-seed-reset');
    if (!seed) return;
    clearSeed(seed);
    const row = r.closest('.nds-custom-brand-row');
    const inp = row && row.querySelector('[data-brand-seed="' + seed + '"]');
    if (inp) inp.value = '#000000';   // native input has no truly-empty state
});
```

## Design choices (resolved)

- **Hue offsets for derived secondary/tertiary:** `+150°` and `−30°`. The +150 avoids the dead-flat 180° complement; −30 gives an analogous accent. Single rule, defensible for any seed.
- **Picker UX:** all three native `<input type="color">` inputs inline; no disclosure for v1. Native picker for shipping speed.
- **Switching away from custom:** leave inline styles and localStorage keys in place — they're inert under any other `data-brand` selector. Returning to custom restores the same palette instantly.
- **Reset button on native color input:** the input has no empty state; reset clears the inline style + storage (so the CSS-derived default takes over) and cosmetically sets the input to `#000000`. Acceptable v1 trade-off; a custom popover would fix the cosmetic mismatch.
- **Browser fallback:** `@supports` block doesn't apply on old engines → user stays on DGA. Picker still updates `--brand-primary` (no JS error) but nothing recolors. By mid-2026 unsupported share is <1%.
- **Neutral ramp + dark-mode overrides + alpha-on-step-500:** skipped for v1 except the one alpha-500-10 patch needed by current footer/components.

## Verification

After building (`bundle exec jekyll serve`) and running `ruby _plugins/js_processor.rb`:

1. **Live update.** Open the topbar switcher → click the Custom row → drag through the primary color picker. Page recolors live (buttons, links, chips, brand surfaces). `document.documentElement.getAttribute('data-brand')` is `"custom"`; `style.getPropertyValue('--brand-primary')` matches the picked value.
2. **Default derivation.** Pick only primary. Inspect any token consuming `--colors-secondary-*` — should be the +150°-rotated derivative. Tertiary likewise (−30°).
3. **Explicit override + reset.** Pick secondary → recolors. Click Reset → secondary returns to the derived default; inline `--brand-secondary` and localStorage key `nds-brand-secondary` both gone.
4. **Persistence across reload.** With Custom active, reload. Zero flash. `<html>` already has `data-brand="custom"` and inline `--brand-*` styles by the first non-empty paint (DevTools Performance → Frames panel).
5. **Switch away + back.** With Custom active → click Crimson → page goes Crimson (inline styles persist, harmlessly). Click Custom → instant return to the same custom palette.
6. **Switch to DGA.** With Custom active → click "Default (DGA)" → `data-brand` attribute removed, DGA palette. Inline `--brand-*` styles persist on `<html>` (harmless).
7. **Fallback.** Toggle the `@supports` rule off in DevTools (or test on a 2023-era browser). Picker still functions, but no token recolor — page stays DGA. No JS error.
8. **Keyboard.** Tab from theme toggle → Custom row → into each color input. Native picker opens with Space/Enter. Reset buttons reachable.
9. **Screen reader.** With Custom active, the row announces "Pick your own, primary"; inputs announce their `aria-label`; `aria-current="true"` on the row.
10. **No shipped-brand regression.** Cycle Crimson/Corporate/Sunset/Foundation Day — unchanged behavior.
11. **Build guard.** `nds-brand.js` remains in the delegated bundle. Confirm `_js/nds-loader.js` still marks `brand` as non-critical (it should — already non-critical today).
